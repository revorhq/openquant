# 30 Days of OpenQuant

A daily challenge that takes you from `pip install` to a zoo-grade
strategy in a month.

Daily prompts live in
[`marketing/content/30-days-of-openquant.md`](https://github.com/revorhq/openquant/blob/main/marketing/content/30-days-of-openquant.md).

## How it works

- 30 daily prompts, 30-90 minutes each.
- Day 1 ships a CLI install + first sync. Day 30 ships a PR to the zoo.
- Post your progress with `#30DaysOfOpenQuant` on X / LinkedIn.
- Finishers get a contributor badge on their GitHub profile.

## Weekly arc

| Week | Theme |
| ---- | ----- |
| 1    | Data correctness — install, sync, walk the HDFC fixture, query DuckDB. |
| 2    | Honest backtesting — cost engine, slippage, walk-forward. |
| 3    | Live-shape — paper broker, compliance, kill switch, notifications. |
| 4    | Ship — design, build, walk-forward, write up, PR to the zoo. |

## Why this works

Public commitment + small daily reps + a clear end artifact. The
content kit gives you the prompts; the project gives you the rails.
