# Good first issues

We curate a steady pipeline of well-scoped tickets labeled
`good first issue`. The goal: a contributor with an afternoon can ship
something real.

## Where to find them

[github.com/revorhq/openquant — `good first issue`](https://github.com/revorhq/openquant/issues?q=is%3Aissue+is%3Aopen+label%3A%22good+first+issue%22)

## What makes an issue good-first

Every good-first issue ships with:

1. **Acceptance criteria** — a checklist of what "done" looks like.
2. **Pointers** — exact files and functions to touch.
3. **A test plan** — the assertion or fixture to add.
4. **Time estimate** — typically 1 – 4 hours.

## Mentorship promise

Comment on the issue saying you'd like to take it. A maintainer will
assign you and check in within 48 hours. You get one review pass per
week, minimum.

## Starter areas

| Area              | Why it's approachable |
| ----------------- | --------------------- |
| Cost-model presets for a new broker | Self-contained file + fixture |
| New cookbook recipe | Pure docs, no public-API risk |
| `oq-data` regression test for a known corporate action | Single test, easily verifiable |
| Tearsheet cosmetic improvements | Localized to one module |
| Translating an existing notebook to the latest API | Mechanical, builds familiarity |

## After your first PR

We add you to the contributors list and ping you for the next cohort.
