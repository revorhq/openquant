# Discord

The full channel layout, role policy, and moderation playbook live in
[`community/discord-channels.md`](https://github.com/revorhq/openquant/blob/main/community/discord-channels.md).

## Channel highlights

- `#announcements` — read-only, releases and outages.
- `#welcome` — new-member intros.
- `#general` — open chat.
- `#data-correctness` — data bugs and golden-dataset discussion.
- `#backtest-honesty` — gross-vs-net reviews, cost-model questions.
- `#broker-help` — Kite/Dhan/Upstox/Fyers integration questions.
- `#mcp-and-llms` — MCP server tooling and LLM-driven workflows.
- `#strategy-zoo` — proposing, reviewing, and discussing zoo entries.
- `#cohort-spring-2026` — active cohort channel (rotates).
- `#good-first-issues` — issue-of-the-week, pair-up requests.

## Norms

- Be specific. "It doesn't work" gets ignored; a repro gets help.
- Don't share signals or tips. Ever.
- Use threads for any conversation longer than three messages.
- Code of conduct applies in DMs as well as channels.
