# SEBI 2026 explainer

The Securities and Exchange Board of India's Retail Algo Framework
becomes fully mandatory on **April 1, 2026**. The short version:

| Requirement                               | How OpenQuant supports it |
| ----------------------------------------- | -------------------------- |
| Every algo order tagged with broker-registered Algo-ID | `StrategyRegistration.algo_id`, enforced on every order |
| Strategy registration metadata            | `StrategyRegistration` dataclass attached to every order |
| Immutable audit trail of all actions      | `AuditLog` — hash-chained JSON-lines |
| Kill switch in every algo system          | `KillSwitch` in framework core; manual + auto-on-max-loss |
| API security                              | Adapters expect user-injected SDK clients; no creds stored by the library |
| Paper-first default                       | `PaperBroker` is the default; live mode is opt-in twice |

## What you still have to do

OpenQuant India provides the *plumbing*. You still need to:

1. Register your Algo-IDs with your broker.
2. Submit your strategy approvals to your broker.
3. Keep your audit log file safe and reviewable for your records.

## When the rules change

Compliance is a versioned module. When SEBI updates the framework,
expect a minor-version bump of `oq-broker` with new fields and
migrations. Subscribe to repo releases.
